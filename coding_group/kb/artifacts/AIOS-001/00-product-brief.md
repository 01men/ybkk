# 00-product-brief.md — 元冰可可 AIOS 产品说明书（初始化棒产物）

> 这份是 orchestrator 在「init」棒落下的产品说明书。它不替代 PRD（那是下一棒），只把用户原始输入固化下来。
> 任何对产品的范围改动必须开新需求（参见 AGENTS.md §7）。

---

## 1. 产品定位

**元冰可可 AIOS** —— 面向**离散制造企业**的 AI 操作系统。核心理念：**不改造现有系统**，直接读取数据元（数据库、Excel、流程规范、会议内容），结合**场景 × 职能的交付标准**，完成**自主业务流程构建**。

**对企业的承诺**：拿到产品 → 直接实现 AI 转型。**不**需要 FDE 调研、不需要业务部门访谈、不需要需求梳理。系统基于数据源和交付标准，自主完成业务链路重构，降低人–人、人–系统之间的摩擦。

## 3. 关键约束（用户在两轮问答中固化）

| # | 约束项 | 取值 | 含义 |
|---|---|---|---|
| 1 | 产品名 | **元冰可可** | 仓库根 `AGENTS.md` 已对齐 |
| 2 | 部署形态 | **私有化** | 企业内网 / 私有云部署；不接 SaaS 多租户 |
| 3 | 数据元输入 | **关系型 DB + 本体论对象数据** | 不止接表，还要把企业对象（设备、物料、工序、人员、客户）建成本体图 |

---

## 4. 行业边界

- **首版覆盖**：离散制造（机械加工 / 装配 / 3C 电子）
- **不做**：流程制造（化工 / 钢铁 / 食品）—— 首版不背这部分语义
- **预留**：未来通过「行业模板包」扩展（流程模板 / 离散子行业模板）

## 7. 下一棒交接

- 接力：`init` → `analyze`
- 派工 Agent：`requirements-analyst`
- 必读：根 `AGENTS.md` §0、§5 + `coding_group/kb/tooling.md` + 3 个 Skill（`prd-template` / `prd-quality-check` / `impact-radius-analysis`）
- 必写：`01-requirement-doc.md`
- 门禁：PRD 自评 ≥ 60（5 维平均：完整性 / 一致性 / 可验证性 / 影响半径 / 行业贴合）
- 通过 → 弹人工关卡（需求确认）
- 失败 → 打回 requirements-analyst，按维度给修复指令

## 8. 用户授权

- 用户授权 orchestrator 在后续阶段**只在强制人工关卡处弹窗**（按 AGENTS.md §3）：

  2. 方案确认（跨模块 / 改接口 → 必弹；首版从零起算，**必弹**）
  3. 验收（必弹）
  4. 连续打回 ≥ 3 次（熔断弹窗）
- 其它阶段不再打扰用户
